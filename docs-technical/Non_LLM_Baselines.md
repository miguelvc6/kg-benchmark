# Non-LLM Baselines

Phase E implements deterministic non-LLM baselines for the fixed Phase C core manifest. These baselines are generated by `src/non_llm_baselines.py` and evaluated through the same `src/evaluate.py`/`guardian.evaluator` path used by reasoning-floor outputs.

## Purpose

The baselines provide paper anchors before prompt development:

- trivial repair-locus diagnosis references;
- a conservative symbolic TypeA proposal baseline;
- a local lookup oracle for TypeB operationality;
- invalid and no-op lower bounds for evaluator sanity.

They are not model runs and do not use provider APIs.

## Command

```bash
UV_PROJECT_ENVIRONMENT=.venv-wsl uv run python src/non_llm_baselines.py \
  --classified-benchmark data/04_classified_benchmark.jsonl \
  --world-state data/03_world_state.json \
  --selection-manifest reports/benchmark_selection/core_v1_seed_13.json \
  --output-dir reports/non_llm_baselines/core_v1_phase_e \
  --tier core
```

The packaged console entry point is:

```bash
UV_PROJECT_ENVIRONMENT=.venv-wsl uv run kg-non-llm-baselines
```

## Implemented Baselines

Track-diagnosis baselines:

- `majority_track`;
- `always_a_box`;
- `always_t_box`;
- `always_ambiguous`.

Proposal baselines:

- `constraint_only_typea`: emits A-box proposals for supported deterministic TypeA repairs and abstains elsewhere.
- `local_lookup_oracle`: emits A-box proposals for TypeB cases when the target value is locally visible or deterministically derivable.
- `do_nothing_pre_repair`: emits the reconstructed pre-repair target-property state for A-box cases.
- `invalid_empty`: writes intentionally invalid raw proposal stubs and evaluator run-manifest parse errors, without feeding malformed proposals into the normalizer.

## Outputs

The default Phase E output directory is `reports/non_llm_baselines/core_v1_phase_e/`.

Top-level outputs:

- `run_config.json`;
- `baseline_summary.json`;
- `baseline_summary.md`.

Each baseline subdirectory writes the relevant artifacts:

- `track_diagnoses.jsonl` for track baselines;
- `a_box_proposals.jsonl` for normalized A-box proposal baselines;
- `raw_invalid_proposals.jsonl` and `run_manifest.jsonl` for `invalid_empty`;
- `evaluation_traces.jsonl`;
- `evaluation_summary.json`;
- `baseline_summary.json`.

## Evaluator Integration

All proposal baselines write normalized A-box proposal JSONL except `invalid_empty`, which records parse errors through the run manifest so the evaluator can score lower-bound behavior without crashing on intentionally malformed inputs.

The evaluator now treats Python-incompatible Wikidata regex patterns as non-matching during supported format-regression checks. This prevents stored Wikidata regex syntax such as Unicode property escapes from aborting baseline evaluation.
